## ðŸŒŸ Features

- **Authentication:** Secure Login and Registration using **Supabase Auth**.
- **Role-Based Access:** Donors, Volunteers, NGOs, and Admins have tailored experiences.
- **Interactive Dashboard:**
    - Live Map Integration for locating nearby needs.
    - Impact Statistics (Meals Served, Volunteers Active).

- **Pages:**
    - **Home:** Landing page with "About Us" and "Our Impact" sections.
    - **NGOs:** Directory of verified partner NGOs with filters.
    - **Contact:** Message submission form (connected to database).
- **Backend:** Scalable database schema on PostgreSQL (via Supabase).

## ðŸš€ Getting Started

Follow these instructions to set up the project locally.

### Prerequisites

- [Node.js](https://nodejs.org/) (v18 or higher)
- A [Supabase](https://supabase.com/) account (Free tier works perfectly)

### 1. Installation

Clone the repository and install dependencies for both frontend and backend.

```bash
# Clone the repository
git clone https://github.com/abhaykatre70/techmentorx-team-abhiyanta.git
cd techmentorx-team-abhiyanta

# Install Frontend Dependencies
cd frontend
npm install

# Install Backend Dependencies (Optional - minimal express server)
cd ../backend
npm install
```

### 2. Supabase Configuration (Critical Step)

To make the app function (Login, Register, Database), you need to set up your Supabase project.

1.  **Create a Project:** Go to [database.new](https://database.new) and create a new project.
2.  **Get Credentials:**
    - Go to **Project Settings** -> **API**.
    - Copy the `Project URL` and `anon public` Key.
3.  **Update Frontend Config:**
    - Open `frontend/src/supabase.js`.
    - Replace the values with your own URL and Key:
      ```javascript
      const supabaseUrl = 'YOUR_SUPABASE_URL'
      const supabaseKey = 'YOUR_SUPABASE_ANON_KEY'
      ```
    - *(Note: The repository might already have valid keys for testing, but it's best to use your own).*

4.  **Create Database Tables:**
    - Go to the **SQL Editor** in your Supabase Dashboard.
    - Copy the SQL script below and Run it to create the necessary tables and policies.

    <details>
    <summary>Click to view SQL Schema</summary>

    ```sql
    -- 1. Create Users Table (extends default Auth)
    create table if not exists public.users (
      id uuid references auth.users not null primary key,
      email text,
      name text,
      role text,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null
    );

    -- 2. Create Messages Table (for Contact Form)
    create table if not exists public.messages (
      id bigint generated by default as identity primary key,
      firstName text,
      lastName text,
      email text,
      message text,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null
    );

    -- 3. Enable RLS
    alter table public.users enable row level security;
    alter table public.messages enable row level security;

    -- 4. Policies
    create policy "Public profiles" on public.users for select using ( true );
    create policy "Users update own" on public.users for update using ( auth.uid() = id );
    create policy "Users insert own" on public.users for insert with check ( auth.uid() = id );
    create policy "Anyone insert msg" on public.messages for insert with check ( true );
    ```
    </details>

5.  **Disable Email Confirmation (Optional for Dev):**
    - Go to **Authentication** -> **Providers** -> **Email**.
    - Uncheck "Confirm email" to allow instant login after registration.

### 3. Running the Application

Open two terminal windows to run both services (or just the frontend if backend logic isn't needed yet).

**Terminal 1 (Frontend):**
```bash
cd frontend
npm run dev
# The app will be available at http://localhost:5173
```

**Terminal 2 (Backend):**
```bash
cd backend
npm run dev
# The API server runs on port 5000
```

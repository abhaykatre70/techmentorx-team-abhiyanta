Helping Hand is a donation and volunteering platform connecting specific needs with willing helpers.

## üöÄ Setup Guide

This project uses **React (Vite)** for the frontend and **Supabase** for Backend-as-a-Service (Auth & Database).

### 1. Prerequisites
- Node.js installed
- A [Supabase](https://supabase.com/) account (Free tier is fine)

### 2. Frontend Setup

1.  Navigate to the directory:
    ```bash
    cd frontend
    ```
2.  Install dependencies:
    ```bash
    npm install
    ```
3.  Configure Supabase:
    - Open `src/supabase.js`.
    - Ensure your `supabaseUrl` and `supabaseKey` are correct.
      *(These are already configured in this repo, but verify if you start a new project)*

### 3. Database Setup (Supabase)

To make the Login and Contact features work, you need to create tables in your Supabase project.

1.  Go to your [Supabase Dashboard](https://supabase.com/dashboard).
2.  Open the **SQL Editor**.
3.  Paste and Run the following SQL to create the required tables:

```sql
-- 1. Create Users Table (extends default Auth)
create table users (
  id uuid references auth.users not null primary key,
  email text,
  name text,
  role text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. Create Messages Table (for Contact Form)
create table messages (
  id bigint generated by default as identity primary key,
  firstName text,
  lastName text,
  email text,
  message text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. Enable Security (Row Level Security)
alter table users enable row level security;
alter table messages enable row level security;

-- 4. Add Access Policies
-- Allow anyone to read user profiles (needed for dashboard display)
create policy "Public profiles are viewable by everyone." on users for select using ( true );

-- Allow users to insert their OWN profile during signup
create policy "Users can insert their own profile." on users for insert with check ( auth.uid() = id );

-- Allow anyone to send a message (Contact Form)
create policy "Anyone can send messages" on messages for insert with check ( true );
```

### 4. Running the App

1.  Start the frontend:
    ```bash
    npm run dev
    ```
2.  Open `http://localhost:5173`.

### 5. Features
- **Login / Register:** Fully functional using Supabase Auth.
- **Contact:** Submits messages to Supabase `messages` table.
- **Dashboard:** Protected route, accessible only after login.
